# Settings

## Overview

Settings is where you manage your workspace: name, branding, regional options, users, teams, roles, API keys, and billing. Access is typically limited to admins or owners. Changes here apply across the whole workspace.

## Navigation

**Sidebar** → User menu (avatar) → **Settings**, or go directly to **/settings** if available in the nav.

## UI Components

- **Tabs** — General, Branding, Users, Teams, Roles, API, Billing.
- **General** — Workspace name, subdomain, timezone, currency, date format.
- **Branding** — Logo, colors (if supported).
- **Users** — Invite, list, deactivate.
- **Teams** — Create teams, add members.
- **Roles** — Permissions and role assignments.
- **API** — Create and manage API keys.
- **Billing** — Plan and subscription (if applicable).

## User Actions

- Update workspace name and subdomain
- Set timezone, currency, and date format
- Upload logo or set branding
- Invite users and manage access
- Create teams and assign members
- Configure roles and permissions
- Create and revoke API keys
- Change plan or billing (if enabled)

For more detail, see **Workspace**, **Users**, **Teams**, **Roles**, and **API** under Settings in the documentation sidebar.

## Best Practices

- Set timezone and currency once so reports and dates are consistent.
- Use roles to limit access (e.g. read-only, per-module).
- Rotate API keys periodically and never share them in code or public repos.
- Keep billing and plan up to date if you use paid features.
