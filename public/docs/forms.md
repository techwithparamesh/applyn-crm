# Forms

## Overview

Forms capture data from your website or shared links and create or update records in the CRM. Use the form builder to add fields, set success messages, and choose the module that receives submissions. Submissions appear as records and can trigger automations.

## Navigation

**Sidebar → Workspace → Forms**. Click a form to edit it in the Form builder.

## UI Components

- **Form list** — All forms with name, module, status (active/inactive), submissions count.
- **Form builder** — Drag-and-drop or list of fields; configure label, required, type.
- **Field types** — Text, email, phone, dropdown, date, etc.
- **Form settings** — Success message, redirect URL, reCAPTCHA, active toggle.
- **Embed / share** — Public URL or iframe/script snippet for your site.
- **Submissions** — List of submissions; open to view or link to record.

## User Actions

- Create a form and select the target module
- Add, reorder, and configure fields
- Set success message and optional redirect
- Activate or deactivate the form
- Copy public URL or embed code
- View and manage submissions

## Step-by-Step Guide

### How to create a form

1. Open **Sidebar → Workspace → Forms**.
2. Click **New form** (or **Create form**).
3. Enter **Name** and select the **Module** that will receive submissions.
4. Add **Fields** (drag or add): label, type, required.
5. Set **Success message** and optional **Redirect URL**.
6. Save and set the form to **Active**; use the public URL or embed code to share.

### How to embed a form on your website

1. Open the form in the **Form builder**.
2. Go to **Share** or **Embed**.
3. Copy the **Public URL** for a link, or the **iframe** / **script** snippet for embedding.
4. Paste into your site; submissions will create records in the linked module.

### How to view submissions

1. Open **Forms** and click the form.
2. Open **Submissions** (or **Responses**); you’ll see each submission with date and data.
3. Click a submission to view details or link it to a record if needed.

## Best Practices

- Map form fields to module fields so data lands in the right place.
- Use a clear success message and optional redirect to a thank-you page.
- Enable reCAPTCHA on public forms to reduce spam.
- Use **Automations** to notify the team or create tasks when a form is submitted.
