# Pipelines

## Overview

Pipelines represent a process with stages—for example, a sales pipeline with stages like New, Qualified, Proposal, Won, Lost. You attach a pipeline to a module (e.g. Deals) and then view and move records through stages on a Kanban board.

## Navigation

**Sidebar → Pipelines**. Select a pipeline from the list; the Kanban board shows records by stage.

## UI Components

- **Pipeline list** — All pipelines; create new or edit/delete existing.
- **Pipeline selector** — Dropdown or tabs to switch between pipelines.
- **Kanban board** — Columns for each stage; cards are records.
- **Stage cards** — Drag and drop records between columns to change stage.
- **Add stage** — Add or reorder stages and set colors.

## User Actions

- Create a pipeline and choose the linked module
- Add, rename, reorder, or remove stages and set stage colors
- View records in Kanban by pipeline stage
- Drag records between stages to update their stage field
- Edit or delete pipelines

## Step-by-Step Guide

### How to create a pipeline

1. Open **Sidebar → Pipelines**.
2. Click **New pipeline** (or **Create pipeline**).
3. Enter a **Name** (e.g. “Sales pipeline”) and select the **Module** (e.g. Deals).
4. Add **Stages** (e.g. New, In progress, Won, Lost) and optional colors.
5. Save; the pipeline appears in the list and can be opened as a Kanban view.

### How to move a record to another stage

1. Open **Pipelines** and select the pipeline.
2. Find the record card on the Kanban board.
3. Drag the card to another column (stage); the record’s stage field updates automatically.

## Best Practices

- Use a small number of stages (e.g. 5–7) so the board stays clear.
- Name stages consistently (e.g. “Won” / “Lost”) so reporting is easy.
- Use the same pipeline across teams that share the same process.
- Combine Pipelines with **Automations** to run actions when a record moves to a stage.
