# Modules

## Overview

Modules are the building blocks of your CRM. Each module represents a type of record—such as Leads, Contacts, Deals, or Companies. You define custom fields and later link modules with Relationships and use them in Pipelines and Reports.

## Navigation

**Sidebar → Modules**. Click a module card to open its record list and views.

## UI Components

- **Module list** — Grid or list of all modules with name, icon, and description.
- **Module card** — Click to open the module; use **Settings** or **Edit** to change name, icon, or fields.
- **Field builder** — Add and configure fields (text, number, date, select, etc.) for a module.
- **Search / filters** — Find modules by name on the Modules page.

## User Actions

- Create a new module
- Edit module name, icon, and description
- Add, edit, reorder, or remove fields
- Open a module to view and manage its records
- Delete a module (when no longer needed)

## Step-by-Step Guide

### How to create a module

1. Open **Sidebar → Modules**.
2. Click **New module** (or **Create module**).
3. Enter **Name** (e.g. “Leads”), optional **Description**, and choose an **Icon**.
4. Save; the module is created with default fields if applicable.
5. Open the module and use **Settings** or **Manage fields** to add custom fields.

### How to add a field

1. Open **Modules** and select the module.
2. Open **Settings** or **Manage fields** for that module.
3. Click **Add field**.
4. Choose field type (text, number, date, dropdown, etc.) and set **Label** and options.
5. Save; the field appears on record forms and in list/table views.

## Best Practices

- Use clear, consistent names (e.g. “Lead”, “Contact”, “Deal”).
- Add only the fields you need to avoid clutter and keep forms fast.
- Use **Relationships** to link modules (e.g. Lead → Contact, Deal → Company) instead of duplicating data.
- Consider installing a **Template** to get modules and fields pre-configured.
