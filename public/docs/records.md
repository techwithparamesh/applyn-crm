# Records

## Overview

Records are the rows in your modules—each Lead, Contact, or Deal is a record. You create, edit, and view records from the module detail page. Record detail pages show all fields, related records, notes, files, emails, and activity.

## Navigation

**Sidebar → Modules → [Select a module]**. From the record list, click a row or **New record** to open the create form or record detail.

**Record detail:** **Modules → [Module] → [Record]** (e.g. Modules → Leads → John Smith).

## UI Components

- **Table view** — Columns for fields; sort and filter.
- **List view** — Compact list of records.
- **Kanban view** — Cards by stage (when a pipeline is linked).
- **Calendar view** — Records by date field.
- **Filters & search** — Narrow by field values or full-text search.
- **Record detail** — Tabs: details, notes, files, emails, related records, activity.
- **Inline edit** — Click a field value to edit in place (where supported).

## User Actions

- Create records (New record)
- Edit records (inline or in detail form)
- Delete records (with confirmation)
- Filter and sort the list
- Switch views (table, list, kanban, calendar)
- Import/export records (CSV)
- Link related records from the detail page
- Add notes, files, and view linked emails

## Step-by-Step Guide

### How to create a record

1. Open **Modules** and select the module (e.g. Leads).
2. Click **New record** (or **+ New**).
3. Fill in the required and optional fields in the form.
4. Click **Save**; the record is created and you can open it to add notes or links.

### How to edit a record

1. Open the module and find the record (search or filter if needed).
2. Click the record to open its **detail** page.
3. Click **Edit** or use **inline edit** on a field.
4. Change values and **Save**.

### How to link a related record

1. Open a record’s **detail** page.
2. Open the **Related records** (or **Relationships**) section.
3. Click **Link record** and choose the relationship and target module.
4. Search and select the record to link; save.

## Best Practices

- Use consistent data in key fields (e.g. status, stage) so filters and reports work well.
- Link records via Relationships instead of pasting IDs or duplicate data.
- Use **Import** for bulk data and **Export** for backups or reporting in spreadsheets.
- Check **Duplicate detection** warnings when creating or editing to avoid duplicates.
